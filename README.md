> **Magic Template** — 템플릿·레퍼런스만으로 에이전트가 이미지를 생성하고, **디테일 편집** 캔버스에서 **직관적인 드로잉**만으로 정교하게 다듬는 **완전 에이전틱(Agentic) 이미지 생성 서비스** (1,275개 템플릿)

#### 🔄 전체 서비스 워크플로우 (End-to-End)

사용자는 **프롬프트를 작성하지 않고** 탐색 → 생성 → 확인 → (선택) 정밀 편집 → 완료까지 한 서비스 안에서 마무리합니다.

```
탐색·선택 → Agentic 생성 → 결과 확인 → (선택) 캔버스 디테일 편집 → 완료 (★영상 확장 예정)
  │             │             │                     └ 직관적 드로잉(영역·화살표)
  └ 템플릿       └ 레퍼런스     └ Before/After          → 에이전트가 정교한 부분 편집 자율 처리
     선택           업로드만       비교·다운로드
```

> **핵심 설계**: Phase 2·4 모두 **완전 Agentic** — 생성은 템플릿·레퍼런스만으로, 편집은 **캔버스 위 직관적 드로잉**만으로 에이전트가 프롬프트 재작성·모델 라우팅·부분 재생성까지 자율 수행하는 **Generate-then-Refine** 구조입니다.

#### 📸 생성 예시 (템플릿 + 레퍼런스 → After)

갤러리에서 **템플릿 이미지**를 선택하고, 사용자가 **레퍼런스(인물/제품) 이미지**를 업로드하면 에이전트가 두 입력을 융합해 최종 **After** 이미지를 자동 생성합니다.

##### 3️⃣ 디테일 편집 (Detail Edit) — 캔버스 드로잉 기반 정밀 후처리

> 템플릿 생성 결과에서 「디테일 편집」을 누르면 Fabric.js **캔버스**가 열립니다. 사용자는 프롬프트나 별도 편집 툴 없이, **영역을 드래그하거나 화살표를 그리는 직관적인 드로잉**만으로 "어디를 어떻게 바꿀지"를 표시합니다. 이후 **캔버스 의도 분석 → 프롬프트 자동 재작성 → 모델 라우팅 → 부분 재생성 → seamless 합성**까지 전 과정을 에이전트가 자율적으로 처리해, 배경·소품·포즈·제품 위치 등을 **정교하게** 다듬을 수 있습니다.

**캔버스 드로잉 → Agentic 편집**
- **직관적 드로잉만으로 정교한 편집**: "왼쪽 상단 배경만 노을로"처럼 글로 설명하기 어려운 수정도, 캔버스 위에서 **영역·화살표를 그리는 것**만으로 충분합니다. Photoshop 없이도 픽셀 단위에 가까운 부분 수정이 가능합니다.
- **시각적 의도 → 구조화**: Magic Canvas에서 검증된 캔버스 분석기가 드로잉(화살표, 원형 마킹, 영역)을 JSON 작업 계획으로 변환하고, 사용자가 쓰지 않은 프롬프트를 에이전트가 대신 작성합니다.
- **정체성 보존**: 인물 얼굴·제품 실루엣 등 `must_keep` 제약을 에이전트가 프롬프트에 자동 반영해, 편집 범위 밖 픽셀은 최대한 유지합니다.
- **멀티 모델 라우팅**: Nano Banana 2/Pro, GPT Image 2 등 작업 유형에 맞는 모델·해상도·비율을 에이전트 파이프라인이 동적으로 연결합니다.

**💡 기존 서비스의 문제 정의 (Problem)**
- **사용자 진입 장벽**: 기존 서비스들은 사용자가 복잡한 프롬프트를 직접 만져야 하거나 특정 프리셋 내로 편집이 제한되어, 비전문가가 커스텀 광고 화보나 프로필 이미지를 제작하기 극히 어렵습니다.
- **레퍼런스 왜곡**: 원본 이미지의 얼굴 형태, 제품 실루엣, 로고 등 핵심 정체성이 생성 과정에서 자연스럽게 유지되지 않고 왜곡되는 현상이 잦습니다.
- **개발/운영 병목**: 템플릿(스타일)이 추가될 때마다 개발자가 매번 개별 매핑 로직이나 전처리/후처리 코드를 수동으로 구현해야 하므로 운영 리소스가 큽니다.
- **생성 후 미세 조정 불가**: 한 번 생성된 이미지에서 특정 영역만 바꾸려면 프롬프트를 처음부터 다시 작성하거나 외부 편집 툴로 넘겨야 하며, 인물·제품 정체성이 깨지기 쉽습니다.

**🛠️ 기술적 해결 및 핵심 성과 (Solution & Value)**
- **Zero-Prompt 파이프라인**: 템플릿과 레퍼런스 이미지를 바탕으로 프롬프트 플래너가 구도, 스타일, 목적을 자율적으로 해석하고 프롬프트를 자동 재작성(Prompt Rewriting)하여 문제를 해결했습니다.
- **O(1) 수준의 무한 확장 아키텍처**: 새로운 비주얼 스타일의 프롬프트와 예시 이미지만 간단히 추가하면 에이전트가 스스로 판단하여 시스템에 자동 적용하므로, **템플릿은 계속 늘어나지만 운영 복잡도는 늘어나지 않는 효율적인 시스템 구조**를 실현했습니다.
- **생성 → 편집 원스톱 경험**: 템플릿 생성 결과에서 바로 **디테일 편집 캔버스**로 전환되는 **Generate-then-Refine** 워크플로우를 구축해, "거의 맞는 결과를 캔버스 위에서 직관적으로 정교하게 다듬기"까지 단일 서비스 안에서 완결했습니다.
- **캔버스 드로잉 기반 정밀 편집**: 영역 드래그·화살표 그리기 등 **직관적인 드로잉**만으로 수정 의도를 전달하면, 에이전트가 캔버스 비전 분석·화살표 관계 해석·프롬프트 재작성·영역 인페인트를 자율 수행합니다.

**🎯 핵심 기능 및 특장점**
- ✅ **기존 서비스의 진입장벽 해소**: 프롬프트 작성·수정 없이 템플릿 선택과 레퍼런스 업로드만으로 커스텀 이미지 생성

### 💄 StylePulse — 뷰티/패션 특화 소비자 여론 및 트렌드 시뮬레이터

> 제품 이미지·UGC 영상과 자연어 지침만으로 온톨로지를 자동 구축하고, 뷰티/패션 특화 가상 소비자 에이전트 군집의 상호작용으로 **소셜 피드 반응·CS 리스크·바이럴 잠재력**을 예측하는 **엔드투엔드 소비자 여론 시뮬레이션 플랫폼**입니다. 동일 코어를 MCP 플러그인(`stylepulse-mcp`)으로 확장해 Cursor/Codex 에이전트 워크플로까지 지원합니다.

#### 🔄 전체 서비스 워크플로우

```
자산 업로드(이미지·영상·카피)
  → Gemini Multimodal + Search Grounding → Neo4j 온톨로지
  → OASIS 페르소나 시뮬레이션(Twitter/Reddit) → SimMemory 누적
  → Graph RAG 리포트 · 1:1 페르소나 인터뷰 · D3 그래프 탐색
  → (MCP) Go/No-Go · 바이럴 예측 · 준수형 대체 시안 생성
```

**🎯 핵심 기능 및 특장점**
- ✅ **멀티모달 자산 → 온톨로지 자동화**: 제품 이미지, 상세페이지, UGC 리뷰 영상의 시각·청각·카피 소구점을 구조화하고 Neo4j 지식 그래프로 변환
- ✅ **Google Search Grounding**: 정식 명칭·브랜드·가격·성분·실시간 평판을 웹 검색으로 역추적해 온톨로지·레퍼런스 카드에 반영
- ✅ **현실 투영형 페르소나**: 피부 타입·퍼스널 컬러 등 뷰티 속성 + MBTI·FOMO·군중 동조·감정 반응성 등 비합리적 소비 심리를 에이전트에 투영
- ✅ **활동성·수면 스케줄링**: `activity_level`·`active_hours`로 심야 비활성 등 실제 라이프사이클을 모사 — 유기적 소셜 반응(팬덤·악플)과 LLM 과금 누수를 동시에 제어
- ✅ **SimMemory (망각·강화)**: 시뮬레이션 전체 발화에 Ebbinghaus 감쇠 + 반복 강화를 적용 — CS 체인은 부정 신호의 `long_term`, 바이럴 체인은 긍정 신호의 `long_term`을 각각 정량 근거로 활용
- ✅ **하이브리드 Graph RAG**: Neo4j 5.x Vector(1536 dim) + Full-Text + Keyword 결합, Lucene 특수문자 파싱 오류 원천 차단
- ✅ **OASIS(CAMEL-AI) 병렬 시뮬레이션**: Twitter/Reddit 병렬 모드 + subprocess·파일 IPC로 웹 서버와 시뮬레이션 엔진 비동기 협업

**💡 기존 서비스의 문제 정의 (Problem)**
- **평면적 여론 분석**: 정형 설문·텍스트 분석만으로는 MBTI·인지 편향·SNS 바이럴 dynamics를 담기 어렵습니다.
- **멀티모달 자산 단절**: 뷰티 화보·숏폼 UGC가 온톨로지·시뮬레이션 환경으로 자동 변환되는 파이프라인이 부재했습니다.
- **실시간 시장과의 괴리**: LLM 학습 Cut-off만으로는 변동하는 가격·성분 트렌드·실시간 평판을 반영한 시뮬레이션이 불가능했습니다.

**🛠️ 기술적 해결 및 핵심 성과 (Solution & Value)**

#### 🔌 seedfit-mcp — 뷰티 D2C 시딩 의사결정 MCP (Cursor / Codex / Grok)

> **제품–크리에이터 적합도 → 포트폴리오 → 브리프 → 성과 학습**을 FastMCP **55 tools (v1.1)** + **`seedfit_ax`** 스킬로 오케스트레이션합니다.  
> 기본은 **1선(추천·집행·벤치마크)**, 인과 OLS/DID는 **2선(데이터 게이트 통과 시만)** — 계량 툴킷이 아니라 **시딩 의사결정 제품**입니다.

**하이브리드 루프 (랭커가 점수, 에이전트가 해석)**

```
제품명 / product_profile (+ budget)
  → (필요 시) 스펙 리서치·입력 정규화
  → hybrid_recommend  ─ v4 matching_score = 기본 진실
        ├ shortlist · safety · 포트폴리오 · 브리프 초안 · decision_log
        └ 순위 변경 시 log_rank_override (사유 필수, 점수 조작 금지)
  → 집행 → 성과 적재 → 벤치마크 회고 → prior/가중치 학습
  → (선택·2선) OLS / DID — 1선 순위를 자동으로 뒤집지 않음
```

| 역량 축 | 무엇을 가능하게 하나 |
|:---|:---|
| **Match** | v4 logistic 0–100 · S~D 등급 · 세이프티(`clear`/`review`/`block`) · 설명 가능 score breakdown |
| **Plan** | 예산·채널·리스크 캡 포트폴리오 · 크리에이터별 시딩 브리프(고지·주의·포맷) |
| **Run & Learn** | 캠페인 상태 머신 · 성과 적재 · ROAS/RPM 벤치마크 · prior·가중치 재학습 |
| **Govern** | override 감사 로그 · PII(Fernet) · health/config · production 안전 가드 |
| **Depth (2선)** | 데이터 준비 시에만 OLS 조건부 연관 · 매칭+DID — **인과 단정 금지** |

**핵심 설계**
- **Hybrid Truth**: 랭커 점수를 에이전트가 재작성하지 않음 — 해석·역할 분담·브리프만
- **Closed-loop**: 시딩 성과가 다음 캠페인 가중치·크리에이터 prior로 환류
- **제품 거리 유지**: 일상 질문은 추천·브리프·벤치마크; 회귀/순증은 게이트 뒤 2선
